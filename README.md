## 🧭 关于我

我是一名计算机相关专业的本科生，平时喜欢做一些能真正跑起来、能展示、也能交付的项目。  
相比单纯堆技术栈，我更关注一个项目背后的 **结构设计、数据流、用户体验和表达方式**。

我现在主要在做这些方向：

- 🧠 **数据结构与算法实践**：用 C++ 做课程设计、刷题系统、推荐逻辑和数据结构应用
- 📱 **移动端应用开发**：使用 Flutter 开发考勤、查课、记录类工具
- 🌾 **机器学习项目**：做过农业病害图像分类、模型微调、验证集评估与可视化
- 🗺️ **专题地图设计**：使用 QGIS 做历史主题地图、路线表达和版面设计
- 🛠️ **后端与部署**：使用 FastAPI、MySQL、Docker、Nginx 等工具搭建服务
- ✍️ **随笔与创作**：写一些带有赛博悬疑、哲思科幻气质的短篇文字
- 🤖 **AI Agent 工作流**：探索 Claude Code、Cursor、OpenClaw 等 AI 辅助开发方式

## 🧩 我正在构建的项目

### 📚 数据结构智能刷题与错题推荐系统

这是一个基于 **C++** 的控制台项目，主要面向数据结构课程设计。

它并不是为了堆前端、数据库或者复杂框架，而是希望把重点放在：

- 数据结构的实际使用
- 刷题记录的组织方式
- 错题本与掌握状态维护
- 基于用户练习情况的题目推荐
- CSV 文件持久化
- 哈希表、堆、图遍历等算法逻辑

这个项目对我来说更像是一次把“课本知识”变成“可运行系统”的尝试。

---

### 📱 考勤助手 App

这是一个基于 **Flutter + FastAPI** 的移动端应用项目，目标是把传统的 Excel 查课、点名、记名流程，整理成一个更清晰的移动端工作流。

目前设计的核心功能包括：

- 点名
- 记名
- 查课记录
- 年级 / 专业 / 班级选择
- 多班级横向切换
- 已到、未到、请假、其他等状态标记
- 周名单提交与审核
- 超级管理员 / 管理员 / 成员权限划分
- 服务端数据同步与统计汇总

这个项目更偏向真实场景应用，也让我接触到了移动端、后端、数据库和权限设计之间的配合。

---

### 🌾 农作物病害识别项目

这是一个图像分类方向的机器学习项目，主要围绕农业病害识别展开。

项目中涉及到：

- 数据集划分与清洗
- PyTorch 模型训练
- ResNet 系列模型微调
- 验证集准确率评估
- Macro-F1 等指标分析
- 混淆矩阵与错误样本可视化

这个项目让我更直观地理解了模型训练不是单纯“跑起来”，还需要关注数据质量、指标选择和结果解释。

---

### 🗺️ 长征主题专题地图

我也在使用 **QGIS** 做专题地图设计，尝试把历史主题、地理空间表达和视觉设计结合起来。

当前做过的主题方向：

> 纪念长征三大主力会师 90 周年专题地图

关注的内容包括：

- 长征路线表达
- 会师地点突出
- 历史节点标注
- DEM / 山体阴影底图
- 地图符号系统设计
- 复古宣传画风格与现代信息图结合

我觉得地图不只是地理信息展示，也是一种叙事方式。
